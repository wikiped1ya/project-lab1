### 5.
Приглашения командной строки задается переменной окружениея PS1. Эта переменная может содержать как обычный текст, так и escape-последовательности, которые преобразуются в определенную информацию. Вот основные escape-последовательности:
* \u — имя текущего пользователя
* \h — имя хоста до первой точки
* \w — полный путь к текущей рабочей директории
* \W — только имя текущей директории
* \t — текущее время
* \d — текущая дата
* \n — символ новой строки

Для отображения текущей ветки Git используется функция __git_ps1. А чтобы она стала доступна, нужно подключить скрипт git-prompt.sh.

## Задание 3 (Эргономика работы с кодом)
### 1.
При помощи команды vimtutor можно изучить команды для vim. Вот наиболее интересные на мой взгляд:
* "o"/"O" - создает новую пустую строку ниже/выше предыдущей и переходит в режим вставки
* "dd" - Удаляет текущую строку
* "yy" - Копирует текущую строку
* "u" - Отменяет последнее изменение
* "gg"/"G" - Перемещает курсор в начало/конец файла
* "Ctrl + r" - Возвращает отмененное изменение

### 2.
* Поиск по содержанию файлов - можно выполнить при помощи команды :vimgrep. Например `:vimgrep /main/ **/*.cpp`
* Перейти к объявлению функции или типа - сначала нужно сгенерировать файл тегов для проекта с помощью команды ctags -R. Затем нужно поставить курсор на имя функции, класса или переменной, нажать Ctrl + ], и vim сразу перейдет к ее определению.
* Перейти к реализации функции или типа - можно использовать поиск по файлам - `:vimgrep /имя_функции/ **/*.cpp`
* Найти все места, где используется переменная, функция или тип - нужно поставить курсор на интересующее слово и выполнить команду `:vimgrep /<cword>/ **/*`
* Переименовать все вхождения переменной, функции или типа - можно скомбинировать команду глобальной замены :s с диапазоном % и флагом c : `:%s/старое_имя/новое_имя/gc`
* Быстро перемещаться между несколькими наиболее интересными местами кода - vim запоминает перемещения, так что можно использовать команду :jumps. Ctrl + o - перейти к более старому месту, Ctrl + i - перейти к более новому месту
* Комментировать и раскомментировать большие части кода - нужно выделить блок текста, нажать : и дописать norm I//, затем нажать Enter
